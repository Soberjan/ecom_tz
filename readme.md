# Структура проекта
В data/ лежит пример с учеником.
В src/ лежит основной код проекта: код для подключению к фастапи и код для работы с базой данных.
В migrations/ лежат миграции, необходимые для создания базы данных.
# Запуск контейнера
Чтобы запустить проект в контейнере, введите команду sudo docker-compose up --build
# Запуск проекта в редакторе кода
1) чтобы установить зависимости, введите pip install -r requiremenets.txt
2) Чтобы подключиться к базе данных, после установки зависимостей введите в консоль команду nox. Там уже прописана команда для поднятия бд из миграции. Возможно, придется вам изменить порт.
3) устанавливать переменные окружения не обязательно, файл .env лежит в репозитории
4) входная точка в проект - файл main.py. Так как он лежит в src, нужно настроить .vscode/launch.json; настроенная версия уже лежит в репозитории.
После этого можно запускать как тесты, так и main.py
# Описание проекта
Проект разбит на модули для того, чтобы его можно было удобно расширять.
api - модуль для работы с фастапи. в нем лежит server/fast.py - небольшой скрипт для запуска сервера. В enpoints лежат аналитические и технические эндпойнты
database - модуль для работы с базой данных. Позволяет считывать данные из DictReader (читалка csv файлов) и вставлять новые записи в таблицу. В папке queries перечислены аналитические запросы в базу данных
config - файл для работы с общими переменными программы, в данном случае, для работы с параметрами подключения к бд
main.py - точка входа в проект
Валидация проводится с помощью Pydantic моделей.
